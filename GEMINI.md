This file provides project-specific instructions to Gemini. All instructions are additive, so instructions in nested GEMINI.md files will be combined with instructions in parent directories. This allows for both global and granular control over Gemini's behavior.

Key areas to customize include:

- **Persona & Tone:** Define how Gemini should communicate (e.g., formal, casual, specific engineering persona).
- **Tech Stack:** Specify frameworks, libraries, and versions to ensure generated code is compatible.
- **Conventions:** Detail coding standards, naming conventions, and architectural patterns.
- **User Preferences:** Note any specific user preferences or workflows.
