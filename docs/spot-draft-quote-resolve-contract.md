# Draft Quote Assistant - Resolve Contract

## Purpose
This document defines the backend-to-frontend contract payload structure for submitting operator decisions made in the Exception Workspace interface. Decisions are posted to the endpoint `POST /api/v3/spot/envelopes/<id>/draft-quote/resolve/`.

## Endpoints

### POST `/api/v3/spot/envelopes/<id>/draft-quote/resolve/`
Submits a batch of human operator decisions to resolve unmapped codes, unclassified segments, or correct faulty amounts.

**Response Statuses:**
* `200 OK`: All decisions validated, applied, and persisted successfully. Returns the resolved status.
* `400 Bad Request`: Validation failure. Payload structure mismatches schema, or proposed codes/values fail verification constraints.
* `401/403`: Unauthorized or Forbidden access.
* `404`: Envelope not found.
* `501 Not Implemented`: Contract defined but persistence logic is pending (current phase behavior).

---

## Root Payload Structure
```json
{
  "idempotency_key": "uuid-v4-client-generated-key",
  "decisions": [
    {
      "decision_id": "dec-uuid-1",
      "type": "accept_suggestion",
      "target_id": "chg-001",
      "details": {},
      "audit_metadata": {
        "user_id": 42,
        "timestamp": "2026-07-03T00:00:00Z"
      }
    }
  ]
}
```

## Decision Types & Details Schema

### 1. `accept_suggestion`
Accepts a system-suggested charge line exactly as it was generated by the intake model.
* **`target_id`**: Inbound `suggested_charges` id.
* **`details`**: Empty object `{}`.

### 2. `map_to_product_code`
Maps a suggested or low-confidence charge line to an existing master ProductCode.
* **`target_id`**: Inbound `suggested_charges` id.
* **`details`**:
  * `product_code`: Master catalog code string (e.g., `"AF-FUEL"`).

### 3. `request_product_code`
Requests the creation of a new ProductCode in the master ERP catalog.
* **`target_id`**: Inbound `suggested_charges` id or unclassified item id.
* **`details`**:
  * `proposed_code`: The recommended code string (e.g. `"AF-WAR"`).
  * `description`: Inferred definition/context.
  * `category`: Categorization class (`freight`, `origin_charges`, `destination_charges`).
  * `domain`: Product domain (`IMPORT`, `EXPORT`, `DOMESTIC`).
  * `reason`: Operator reason for the request.

### 4. `ignore`
Intentionally ignores/excludes a charge line or unclassified block of text.
* **`target_id`**: Inbound `suggested_charges` or `unclassified_items` id.
* **`details`**:
  * `reason`: Explaining why this item is non-commercial or boilerplate (must not be empty).

### 5. `edit_charge`
Corrects incorrect values (amounts, units, currencies) extracted from the document.
* **`target_id`**: Inbound `suggested_charges` id.
* **`details`**:
  * `original_values`:
    * `amount`: Original numeric amount.
    * `currency`: Original currency code.
    * `rate`: Original rate (if any).
    * `unit`: Original unit (if any).
  * `updated_values`:
    * `amount`: Corrected numeric amount.
    * `currency`: Corrected currency code.
    * `rate`: Corrected rate.
    * `unit`: Corrected unit.

### 6. `classify_unclassified`
Maps an unclassified block of text to a structured charge line.
* **`target_id`**: Inbound `unclassified_items` id.
* **`details`**:
  * `bucket`: Target bucket (`airfreight`, `origin_charges`, `destination_charges`).
  * `display_label`: Descriptive label.
  * `product_code`: Existing master code string or requested proposed code.
  * `amount`: Numeric value.
  * `currency`: 3-letter currency code.
  * `rate`: Inferred rate (optional).
  * `unit`: Inferred unit type (optional).
  * `minimum_charge`: Inferred minimum limit (optional).
